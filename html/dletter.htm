A Message from the HighWire Development Team

Welcome to the latest release of HighWire, version 0.2.3

Even though it has not been that long since the last release of HighWire, the recent changes seems to suggest that it's already time to make another public release available. The useability of HighWire has now been significantly increased due to the implementation of imagemaps. This means that all those sites you have been unable to utilize due to their site navigation being hidden in imagemaps are now likely to be working just fine!

Another improvement regarding site navigation and functionality is that HighWire now can handle having images working as form elements. When it comes to the user interface HighWire is now also better at handling popup menus that are too long to fit the desktop height. If such a menu is opened now, you will notice that there are scroll buttons added to the menu in order to allow you to scroll to the part of the popup menu that is currently located outside the visible screen area. A small bugfix has also been made to text input fields, were HighWire earlier on has been overriding the document when it tried setting a field width below 20 characters.

HighWire has also extended its support for image formats since 0.2.2 release. The new formats XBM and XPM are rarely used these days but are still in use by servers like eg. NCSA. Furthermore, all other decent browsers does support at least XBM. Even CAB does actually, and since HighWire is setting out to be a replacement for CAB it seemed obvious we should support these formats too!

As usual, more details of changes can be seen in the Change.Log file.

– And as always for your help and support, Thank you,

– HighWire Development Team
