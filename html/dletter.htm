A Message from the HighWire Development Team

Welcome to the latest release of HighWire, version 0.2.2

    We hope that you will enjoy this version as much as we have in the development stage.

– And as always for your help and support, Thank you,

– HighWire Development Team

Current Release Changes:

  • Large portions of the cache handling code reworked. This gets rid of several wrong warnings about not found files (mostly due to login pages for forums).
  • Image width/height are saves in the disk cache now, this reduces reformat flickering on re-visited pages to the nearly absolute minimum (#0133).
  • More types of corrupted jpeg image data are catched now, instead of aborting.
  • The window handling code also got a large rework to have non-modal dialog windows in the future. As a little gimmick the F10 key does the same now as the F11 does in Mozilla/IE.
  • Improved detection of system type (AES + kernel), now XaAES is recognized also. This also allows to get rid of the window fuller bug with XaAES (#0159, #0167).
  • Additional filtering method to detect images which has a particulare directory name in their URL. This method is used if the search pattern starts with a '/'. For example.
       RESTRICT_HOST I /ads/
    matches all images which has '/ads/' somewhere in their path.
  • CTRL-V can be used now to paste text from the clipboard into formular input fields.
  • Bugs #0133, #0148, #0150, #0167, #0169, #0170, #0176, #0177 resolved.
  • ... and several more to be found in the Change.Log.

As a reminder, don't forget to check the example file for explanations of new configuration keywords from the last release!
