A Message from the HighWire Development Team

Welcome to the latest release of HighWire, version 0.2.0

    If you are asking yourself what happened to versions 0.1.8 and 0.1.9, don't worry you did not miss them. We felt that this version was such a major upgrade over the last version that we bumped the version number up to 0.2.0, completely bypassing .1.8 and .1.9. Sometimes the recognition sinks in that a project is maturing. After you use this release for awhile, we think you will agree with us.

    As you cruise around you will notice that Form support is in place. Google away my friend, or pop over to the HighWire Forum (handily linked in the left frame) and drop us a note. Some user interface extensions have been added for saving content and grabbing those handy URL's to the clipboard to send to your friends on AtarIRC or AtarICQ or just to save out to an email.

    Then of course we have this whole new document format. Long gone are the blue frames with yellow borders put together on one afternoon to demonstrate the capabilities of HighWire. HighWire is growing up and it was time for it's files to grow up as well. You should find this version to not only look a bit more polished, but to be a bit more helpful. Click on the Documentation link in the left frame to take a look at the new docs. And if you don't have online access and want to take highwire for a few tests, the demonstration html files are now located under the HTML Test Suite link in the left frame.

    We hope that you will enjoy this version as much as we have in the development stage.

– And as always for your help and support, Thank you,

– HighWire Development Team

Current Release Changes:

  •  Pop-up menu with Save As... and Copy URL added
  •  <TEXTAREA> tag implemented.  The length of the typed in text is restricted only by the system RAM 
  •  Keyboard input implemented for form tags of the type <INPUT type=text ..>.  Jumping to other text fields of a formular can be done by the TAB key, inclusive automatical page scrolling to the reached field.  Leaving the edit mode can be done with CTRL+TAB.
  •  Table column width calculation changed, was causing crashes sometimes.
  •  Speed up CSS parsing by avoiding useless CSS interpretation for closing tags, saves up to 20% parsing time on some sites.
  •  Cache improved to avoid storing of contents resulting from POST actions.
  •  Handling of form method POST implemented.

See the Change.Log file for the full list of updates.

As a reminder, don't forget to check the example file for explanations of new configuration keywords from the last release!
